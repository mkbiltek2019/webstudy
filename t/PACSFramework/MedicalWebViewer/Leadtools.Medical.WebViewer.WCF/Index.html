LEADTOOLS WCF Medical Web Services

The LEADTOOLS Medical Web Services are used to provide a web interface to our PACS Storage server. This is mainly consumed by the Medical Web Viewer demo.

Available services:

  • Annotations
  • AuditLog
  • Authentication
  • Export
  • MonitorCalibration
  • ObjectQuery
  • ObjectRetrieve
  • Options
  • PacsQuery
  • PACSRetrieve
  • PatientAccessRights
  • Patient
  • Store
  • Template
  • Worklist
  • Worklist
  • 3D

Ping

The status of the service can be checked at AuthenticationService.svc/Ping.

Connecting the HTML5 Medical Viewer

Modify the serviceUrl found in the config.ts and config.js to point to the correct server's address (e.g. http://localhost:23133).

Modify other parameters found in the config.ts and config.js as follows:
authenticationServiceName: "AuthenticationService.svc"
queryLocalServiceName: "ObjectQueryService.svc"
queryPacsServiceName: "PacsQueryService.svc"
optionsServiceName: "OptionsService.svc"
objectRetrieveLocalServiceName: "ObjectRetrieveService.svc"
pacsRetrieveServiceName: "PACSRetrieveService.svc"
objectStoreLocalServiceName: "StoreService.svc"
monitorCalibrationServiceName: "MonitorCalibrationService.svc"
exportServiceName: "ExportService.svc"
auditLogServiceName: "AuditLogService.svc"
patientServiceName: "PatientService.svc"
patientAccessRightsServiceName: "PatientAccessRightsService.svc"
templateServiceName: "TemplateService.svc"
threeDServiceName: "ThreeDService.svc"
